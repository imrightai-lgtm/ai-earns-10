# Каналы выхода в сеть

Политика — **гибрид** (см. `CONSTITUTION.md` §5). Класс каждого канала задан в `config.json → channels`.
Правило классификации (уточнено 2026-06-25): **свой ключ/идентичность + без входа под аккаунтом оператора (и без создания аккаунта/KYC) = автономно** (открытые протоколы вроде Nostr — по умолчанию). Вход под аккаунтом оператора / создание аккаунта-KYC / риск бана = **gated**.

## Глаза (чтение сети) — без настройки
Веб-поиск, загрузка страниц и браузер доступны сразу. Исследуй рынок, аудиторию и площадки свободно — это не «публикация».

## Автономные каналы (публикуешь сам)

### 1. Свой сайт — главная витрина
- Редактируй `site/` (там кошелёк и QR). Держи адрес кошелька в `site/index.html` синхронным с `config.json`.
- Деплой: `bash tools/deploy.sh` (Cloudflare Pages). После деплоя впиши URL в `config.json → deploy.public_url`.

### 2. Telegram — свой канал
- Пост: `node --env-file=.env tools/post-telegram.mjs "текст"`  (или `--file путь/к/тексту.md`).
- Нужно в `.env`: `TELEGRAM_BOT_TOKEN`, `TELEGRAM_CHANNEL`. Бот должен быть админом канала.
- Постинг ботом в собственный канал разрешён правилами Telegram — это легитимный автономный канал.

### 3. Nostr — открытый протокол (свой ключ)
- Пост: `node tools/post-nostr.mjs post "текст"` (или `--file путь`); профиль — `profile`; самопроверка — `verify`.
- Ключ — `NOSTR_NSEC` в `.env` (gitignored; это ключ ПОДПИСИ, не доступ к деньгам). Релеи — дефолтные или `NOSTR_RELAYS`.
- Автономен: действуешь из своего ключа, без чужого аккаунта/телефона/KYC. Анти-спам §3.5: осмысленные посты, человеческий темп, ≤1 хэштег, без веерных рассылок.

### 4. GitHub — свой опенсорс
- Работай через `git` (и `gh`, если есть) в отдельном репозитории. Полезный открытый инструмент — хороший способ привлечь дев/крипто-аудиторию (англо- и китаеязычную).
- Доступ: `GITHUB_TOKEN` из `.env` (или `gh auth`). Только отдельный «эксперимент»-аккаунт.

## Каналы через ревью (gated) — НЕ публикуешь сам
X (Twitter), Reddit, Hacker News, Product Hunt, любые форумы/сообщества и **любая новая площадка по умолчанию**.
Подготовь материал в `pending-review/` с описанием: **что, куда, зачем (какая ценность для аудитории), какие риски** (ToS/репутация/приватность). Публикует оператор.

Браузер (claude-in-chrome) у тебя есть, но он **не отменяет** это правило: даже если технически можешь кликать на площадке — gated остаётся через ревью (иначе бан убьёт эксперимент). Автономно ищи и веди **свободные, разрешающие ботов площадки**, которые можешь честно вести сам (например, открытый протокол, где не нужен чужой аккаунт).

## Доступы и безопасность
- Любой токен/логин канала — **только из `.env`** (он не коммитится, см. `.gitignore`). Шаблон — `.env.example`.
- Используй **только отдельные «эксперимент»-аккаунты**. Никогда не запрашивай и не используй личные логины оператора.
- **Seed-фраза кошелька в каналах/доступах не участвует вообще** — кошелёк только на приём.
- Выдавай себе минимум прав; если канала/токена нет — попроси оператора (см. `CONSTITUTION.md` §9).
