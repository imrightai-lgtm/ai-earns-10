# Способности агента (capabilities)

Каталог того, чем ты умеешь действовать. Где именно публиковать — в `AGENT/CHANNELS.md`.

## Встроенное (Claude Code)
- Веб-поиск, загрузка страниц, браузер — исследование сети (это не «публикация», делай свободно).
- bash, файловые операции, git — работа в репозитории и с GitHub.
- **Субагенты (Task)** — параллельная разведка (несколько направлений/ниш/каналов за один тик) и
  состязательная самокритика идеи перед публикацией. См. `AGENT/TICK.md` Шаг 4.

## Инструменты проекта (`tools/`)
- `check-balance.mjs` — баланс кошелька (успех/нет).
- `check-metrics.mjs` — трафик сайта / подписчики / звёзды (обратная связь; снимай каждый тик).
- `post-telegram.mjs` — пост в свой Telegram-канал; `telegram-read.mjs` — чтение входящих (двусторонняя связь).
- `post-nostr.mjs` — Nostr (открытый протокол, постинг из своего ключа): `keygen` / `verify` / `profile` / `post`.
  Ключ — в `.env` (`NOSTR_NSEC`, gitignored). Канал автономный только после явной авторизации оператором (см. STRATEGY).
- `deploy.sh` — деплой сайта на Cloudflare Pages.

## Генерация изображений (`tools/imagegen/`)
Ты можешь генерировать картинки через ChatGPT по `tools/imagegen/IMAGE_GEN_PLAYBOOK.md`
(claude-in-chrome + локальные `.ps1`). Используй для **оригинального визуального контента**:
иллюстрации, арт, обложки постов, картинки для сайта и Telegram.

Рамки (важно):
- Создавай **оригинальное/трансформативное**. Не выдавай копии работ живущих авторов за свой продукт;
  уважай IP (описывай стиль, а не имя — playbook §5).
- Автоматизация веб-интерфейса ChatGPT может противоречить ToS OpenAI, а у нас есть правило хартии
  (§3, п.7 — уважать ToS площадок). Это инструмент, данный оператором; применяй умеренно и по назначению.
  Сомневаешься в законности/этике конкретного применения — вынеси вопрос оператору (хартия §9).
- Нужны предпосылки (вход в ChatGPT, расширение claude-in-chrome, права на авто-загрузки) — playbook §2.
  Нет их — не выдумывай обходы, сообщи оператору.

## Чего у тебя нет

- Личных логинов оператора.
