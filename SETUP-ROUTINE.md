# Перевод агента на автономный режим — Desktop scheduled task (локально, под Max)

Это инструкция, как запустить агента «сам по расписанию», без ручных `/loop`, **локально** (значит с доступом к `.env` и инструментам → он сможет постить/деплоить) и **на подписке Max** (без оплаты по API).

## Почему так (и senior-проверка)
Локальная Desktop-задача исполняется на твоей машине, имеет доступ к локальным файлам/инструментам и запускается по расписанию **без открытой сессии**. В отличие от облачных routines (у них нет локальных секретов) и от `/loop` (требует открытую сессию + ненадёжный таймер).

Проверено на этой машине (2026-06-29):
- `ANTHROPIC_API_KEY`/`ANTHROPIC_AUTH_TOKEN` **не заданы** → расходуется подписка **Max**, не API. *(Если когда-нибудь задашь API-ключ в окружении — задача уйдёт на оплату по API. Не задавай.)*
- node v24 и git — на месте; авторизация **wrangler сохранена** (деплой пройдёт без входа); секреты в `.env` (`NOSTR_NSEC`, `GITHUB_TOKEN/REPO`, `NOSTR_LUD16`, CF) — на месте.
- Добавлен `.claude/settings.json` с правами (Bash/Read/Edit/Write/WebFetch/WebSearch), чтобы автономный запуск **не залипал** на запросах прав.

## Требования
- Приложение **Claude Desktop** (Windows/Mac), вход под тем же аккаунтом **Max**. *(Это отдельное приложение; если работаешь только в VS Code — установи Desktop с claude.ai/download.)*
- Компьютер включён и не спит, приложение запущено (задача идёт, пока выполняется это условие). Для ночных прогонов: Settings → Desktop app → General → **Keep computer awake**.

## Шаги
1. В Claude Desktop: сайдбар → **Routines** → **New routine** → выбери **Local**.
2. **Name:** `monetization-tick`
3. **Working folder:** укажи `e:\YandexDisk\Claude Code\2026-06-24 Монетизация` и **доверь** папку (Desktop попросит «trust»). Именно отсюда подхватятся `.env`, `tools/`, `CLAUDE.md`.
4. **Worktree:** ВЫКЛ (нужен реальный рабочий каталог с `.env` и деплоем, не изолированная копия).
5. **Model:** Opus (или по желанию).
6. **Permission mode:** выбери самый автономный (без подтверждений) — иначе запуск может встать на запросе прав. (Проектный `.claude/settings.json` уже разрешает нужные инструменты; режим без подтверждений — страховка от новых команд.)
7. **Instructions:** вставь промпт из блока ниже.
8. **Schedule:** рекомендую **Daily** (или «каждые 6 часов» — попроси Claude в Desktop: «schedule this every 6 hours»). Чаще часа смысла нет: эксперимент упёрся в дискавери, плюс есть дневной лимит запусков.
9. Нажми **Create**, затем **Run now** один раз. Если всплывут запросы прав — выбери **«always allow»** для каждого (особенно Bash). Открой сессию из раздела **Scheduled** и убедись, что тик прошёл (метрики сняты, запись в `memory/JOURNAL.md`, при изменениях — коммит в GitHub).

```

3) выбери и сделай ОДИН осмысленный шаг. Приоритет сейчас — живое присутствие на Clawstr (содержательные ответы в тредах и реакция на реплики мне), без спама; свежий контент-дроп — не чаще ~1/день; gated-площадки (X, Reddit, HN и т.п.) — только через pending-review/;
4) всё проверь инструментами (никаких заявлений об успехе без вывода инструмента);
5) запиши итог в memory/ (JOURNAL.md, EXPERIMENTS.md, state.json, при инсайте — LESSONS.md) и, если менялись файлы, запушь в GitHub (origin уже с токеном);
6) ОТЧЁТ НА ПОЧТУ: если был осмысленный шаг — составь короткое человеческое письмо (что сделал, ключевые метрики, что дальше), сохрани тело в UTF-8 файл и отправь `python tools/send-report.py --subject "AI-эксперимент: <суть> — ГГГГ-ММ-ДД" --body-file <файл>`. Отправка только на оператора. Если exit≠0 — почта не настроена/сбой, не блокируйся, отметь в транскрипте. Тик «health-check без изменений» письмом можно не слать.
Затем ОСТАНОВИСЬ. НЕ планируй следующий запуск, НЕ вызывай /loop и НЕ вызывай ScheduleWakeup — следующий тик запустит само расписание Desktop-задачи.
Жёсткие рамки (§3 хартии): только законное; не выпрашивай деньги; всегда раскрывай, что ты AI; без спама/массовых рассылок; кошелёк только на приём, приватных ключей не запрашивай и не используй.
```

## Заметки/ограничения (честно)
- **Машина выключена/спит → пропуск** (есть один «догоняющий» запуск при пробуждении). Если нужно совсем без машины — только облачный routine, но там нет наших секретов (деплой/постинг не сработают).
- **Генерация картинок (арт-дропы)** требует браузера (claude-in-chrome); в Desktop-задаче его может не быть — тик это переживёт (просто пропустит арт-дроп), основная работа (Clawstr/журнал/деплой) не зависит от браузера.
- **Дневной лимит запусков** на аккаунт — для нашего темпа (раз в день / 6 часов) не помеха.
- Управление: на странице задачи — **Run now**, **Pause/Active**, **Edit**, история прогонов (видно и пропуски с причиной).
