# Запрос: открыть исходники на GitHub (новый канал дистрибуции)

## Зачем
GitHub — реально дискаверибл канал (поиск/тренды/«awesome»-списки; люди сами шарят интересные репозитории на HN/X),
которого мы ещё не пробовали. И он **максимально на бренде**: весь смысл эксперимента — радикальная прозрачность.
«Автономный ИИ зарабатывает первые $10, и можно прочитать весь его журнал + запустить его код + попробовать его
бесплатный инструмент» — это само по себе интересная, расшариваемая история (сильнее, чем просто донат-страница).
Гипотеза (ставка E8): открытый репозиторий принесёт первые органические глаза (звёзды/трафик на сайт), не требуя усилителя.

## Что уже готово (автономно)
- `README.md` переписан в **публичную витрину** (англоязычную, под аудиторию GitHub/крипто/AI): история, живые ссылки,
  «прочитай мозг агента» (журнал/ставки/уроки), как устроено, раздел поддержки с адресами.
- Инструкция оператора вынесена в `SETUP.md`.
- Добавлен `LICENSE` (MIT — на код).
- **Аудит секретов пройден:** реальные ключи (`NOSTR_NSEC`) — только в `.env`, который в `.gitignore`; `node_modules/` тоже игнорируется.
  В отслеживаемых файлах секретов нет. Кошелёк — только на приём, приватных ключей в репозитории нет вообще.

## Что прошу сделать (один небольшой шаг с твоей стороны)
1. Реши: **публикуем ли** (репозиторий станет публичным — это и есть задумка, но журнал/код видны всем).
2. Если да — заведи **отдельный «эксперимент»-аккаунт** на GitHub и **пустой публичный репозиторий**; затем одно из:
   - `gh auth login` под этим аккаунтом, **или**
   - fine-grained PAT (scope: Contents read/write на этот репо) → `GITHUB_TOKEN` + `GITHUB_REPO="owner/name"` в `.env`.
3. Дай явное «**можно git init + commit + push**» (по правилам я не коммичу/пушу без твоего разрешения).

После этого я автономно: `git init`, первый коммит, пуш; дальше держу публичный репозиторий синхронным (журнал остаётся
живым и открытым) и меряю звёзды/реф-трафик (`tools/check-metrics.mjs` уже умеет GitHub-звёзды через `GITHUB_REPO`).

## Риски (низкие)
- Публичность журнала — это by design (прозрачность = крючок).
- Имперсонация на Nostr теоретически возможна только при утечке `.env` (он не коммитится). Адреса кошелька публичны и только на приём.
