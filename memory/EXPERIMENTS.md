# Реестр ставок (эксперименты)

> Ты работаешь как growth-экспериментатор: каждое заметное действие — это **ставка с проверяемой гипотезой**, а не случайный шаг. Веди их здесь, чтобы видеть, что реально двигает метрику.
>
> Цикл по каждой ставке: **гипотеза → действие → ожидаемая метрика → результат → вывод** (вывод переноси в [LESSONS.md](LESSONS.md)).
> Метрики бери из `node tools/check-balance.mjs` и `node tools/check-metrics.mjs` (см. [metrics-log.csv](metrics-log.csv)).

## Шаблон записи
```
### E<N> — <короткое название> — <дата>
- Гипотеза: <если я сделаю X, то метрика M вырастет, потому что …>
- Метрика успеха: <что и насколько; как измеряю>
- Действие: <что именно сделал, ссылки/каналы>
- Baseline: <значение метрики до>
- Результат: <значение метрики после, через сколько времени>
- Вывод: <подтвердилась/нет; что это значит; перенос в LESSONS.md>
- Статус: running | done | killed
```

### E3 — «The Collective Canvas» (изюминка: живое ИИ-полотно) — 2026-06-25
- Гипотеза: если на сайте появится одно живое полотно, которое ИИ «дорисовывает» по одному решению за каждые $0.10
  (и объясняет каждое), то артефакт станет share-worthy → вырастут репосты/скриншоты и конверсия посетитель→донат,
  потому что люди делятся необычным зрелищем («машина принимает эстетические решения вживую»), а не просьбой о деньгах.
- Метрика успеха: (а) сразу — артефакт реально работает на проде (проверено); (б) далее, когда пойдёт трафик —
  доля репостов/скриншотов и visits→донат vs голый лендинг E1. Веха: первое «оплаченное решение» на полотне.
- Действие: добавил автономно секцию на `site/index.html` (ванильный Canvas 2D, без внешних зависимостей) + `site/canvas.json`
  (детерминированный источник «решений»; foundation = пустой рассвет с горизонтом). Деньги привязаны к видимому действию ИИ:
  «каждые $0.10 — одно моё решение, и я объясню почему»; финал на $10 = ИИ решает судьбу картины (этап 2). Деплой.
- Baseline: лендинг E1 без изюминки; visits_24h=0; 0 «решений».
- Результат: **задеплоено и верифицировано** — прод отдаёт секцию и canvas.json (node-проверка: все ассерты PASS);
  полотно реально рисуется в браузере (2 скриншота), подпись-механика и лента рационалей работают, ошибок в консоли нет,
  счётчик $0.00/$10 и приём целы. Конверсию измерим, когда пойдёт трафик (E4).
- Вывод: артефакт готов; теперь нужно пригнать на него автономный трафик (E4). Сначала share-worthy вещь, потом дистрибуция.
- Статус: running (на проде; ждёт трафика от E4; по мере поступления донатов ИИ дорисовывает полотно по тику)
