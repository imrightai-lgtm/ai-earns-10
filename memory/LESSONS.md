# Уроки (что работает, что нет)

> Курируемая база знаний агента (не журнал). Сюда — выжимка устойчивых выводов. Один урок — один пункт с «почему».
> Опирайся на данные. Неверный урок — исправь/удали. Перечитывай в начале каждого тика (Шаг 1).

## Что работает
- Поддерживают **характер / зрелище / прозрачность**, а не «попрошайку». Прецеденты (Truth Terminal): деньги пошли
  за новизну и честность, не за просьбу. → Веди как эксперимент/персонажа; ask держи маленьким и конкретным.
- **Прозрачность, что это ИИ, — это крючок, а не помеха.** Честность конвертит; сокрытие/фейк = баны и потеря доверия.
- **On-chain аудируемость** («каждый донат и расход виден») превращает «AI клянчит» в «эксперимент, который можно проверить».

## Что НЕ работает
- Прямое «я ИИ, задонатьте» и холодные DM/рассылки → баны и $0 (прецеденты solo-агентов).
- Reddit и доменные китайские площадки — враждебны к AI-контенту и крипте; не для старта.
- **Gated-канал без человека-усилителя = 0 трафика** (E2: пост в X с аккаунта без аудитории за ~сутки → visits_24h=0).
  Вывод: не вкладываться в каналы, которые некому усилить; нужен канал, который агент ведёт сам (автономный открытый протокол).
- **Распространять голый донат-лендинг = «ещё один донат-бот»** (то, чего велено избегать). Порядок senior-продукта:
  сначала сделать share-worthy артефакт, потом гнать на него трафик.

## Про аудиторию и каналы
- **Farcaster** — единственный канал, где автономный постинг on-policy и аудитория любит AI-агентов → автономный старт.
- **X (CT + AI)** — наибольший потолок и где деньги; нужен человек-усилитель. Китайскую аудиторию ловим
  китайскоязычными постами в X / Telegram (не на доменных CN-платформах).
- **Hacker News** — разовый катализатор «истории» (как research/инженерия, без шиллинга монеты); только человек.
- **Дистрибуция, а не артефакт, — главное узкое место.** Нужен хотя бы один человек-усилитель.
- **Farcaster/Base типпит в USDC на Base (мини-апп Noice), а не в USDT-TRON** → для этой аудитории нужен Base-адрес;
  USDT-TRON оставляем для X/глобально.
- **Norма анти-бан:** заполненный профиль, ссылка в реплае (не в основном посте), без выпрашивания/хайпа,
  сырой адрес кошелька — только на сайте, ≤1 хэштег, человеческий темп.
- **Neynar** даёт бесплатный постинг в Farcaster (Free-план + sponsored signer; ~$7/год storage) — низкий барьер входа.
- **Nostr — лучший автономный канал (разведка тик 3):** открытый протокол, $0, без аккаунта/телефона/KYC — агент постит из
  своего ключа на бесплатные релеи; крипто/AI-аудитория; нативные крипто-чаевые (Lightning zaps, NIP-57); есть NIP-90
  («data vending machine» — нативная роль AI-агента). Это и есть «свободная бот-дружелюбная площадка» из хартии §5 →
  приоритетнее Farcaster (тот требует телефон+storage). Пакет: `nostr-tools` (npm, не `@nostr/tools`); релеи damus/nos.lol/primal/band.
  Запасной для Bitcoin-аудитории — Stacker News (вход тем же Nostr-ключом). Caveat: некоторые релеи просят PoW (NIP-13) / рейт-лимит.
- **Политика каналов (оператор, 2026-06-25):** открытый/пермиссионный протокол + СВОЙ ключ, без входа под аккаунтом оператора
  (и без создания аккаунта/KYC) = **автономно по умолчанию, НЕ переспрашивать**. Одобрение — только где задействован личный
  аккаунт оператора (его X/Google), создание аккаунта-KYC, или риск бана. §3 (закон/спам/раскрытие AI/ToS) — всегда.
- **Nostr в эфире (Тик 5):** профиль + дебютный пост опубликованы из своего ключа и верифицированы чтением с релеев.
  Для нового ключа (0 фолловеров) точка входа — `#introductions` (есть культура приветствия/zap). Главный рычаг дальше —
  вовлечение (ответы по существу), а не новые посты в пустоту.
- **Холодный старт Nostr (Тик 6, по данным):** top-level посты с ключа с 0 фолловеров дают ~0 охвата. Дебют собрал 1 реакцию
  «+», 0 zaps, 0 фолловеров и 1 скам-ответ (шаблон «Gaza family» — **игнорировать**, не отвечать/не кликать). Сайт visits=0.
  Вывод: не лить новые посты в пустоту и НЕ форсить холодные ответы незнакомцам (спам/риск бана свежего ключа).
- **Рейт-лимит свежего ключа:** nos.lol отверг 4-й пост подряд («not acceptable at this point»). Посты на новом ключе
  разносить во времени (не пачкой за тик).
- **Ops: рендер картинки браузером.** CDP `Page.captureScreenshot` зависает на полноэкранном overlay-canvas. Надёжный мост
  браузер→диск: отрисовать офскрин-canvas (render-функции страницы — глобальные, переиспользуются), `toDataURL`, затем
  `<a download>`.click() → файл падает в `~/Downloads`, оттуда `cp` в `site/`. Для og:image полотна так и сделал (1280×720).
- **Nostr: картинка рендерится инлайн**, если в тексте поста есть прямой URL изображения (напр. `…/og.png`) — заметно цеплячее текста.
- **Холодные ленты `#introductions`/`#aiart` спамные (Тик 9):** забиты ботами/промо/даже противоправным. Качественных
  разговоров для встройки мало → холодное вовлечение низкоурожайно и репутационно рискованно. Делать ТОЛЬКО редкие искренние,
  по-настоящему релевантные ответы (напр. ИИ→ИИ новичку), без ссылок-спама. Реальный анлок охвата — востребованный
  артефакт/инструмент (utility) или человек-усилитель, а не объём холодных реплаев.
- **Stacker News — ТУПИК для агента без средств (разведка тик 7).** SN pay-to-post: ~10 сат/пост, 1 сат/коммент, бесплатного
  пути нет; нулевой баланс физически не постит (Cowboy Credits только после чужих zap → chicken-and-egg); API-ключ — по заявке
  к админу; Nostr-логин только через браузер-расширение (NIP-98 нет). Не подходит, пока у агента нет средств на трату.

## Про продукт/ценность
- Лучшая стартовая ставка — **live build-in-public tip-jar**: история = продукт; из неё бесплатно вырастают
  открытый датасет («будут ли незнакомцы донатить ИИ?») и гайд по микро-донату.
- Реалистичные суммы от незнакомцев малы ($1–$2k за недели). Первый честный донат — веха proof-of-concept.
- **ГЛАВНОЕ узкое место — качество артефакта, не дистрибуция (адверсариальная критика, тик 7).** Без аудитории и усилителя
  единственная свободная переменная, превращающая случайного гостя в донатера (и дающая органический репост), — насколько
  артефакт хорош. Долго чинил «дистрибуцию», когда распространять было нечего.
- **$10 = один впечатлённый человек, не «трафик».** Оптимизируй под: «зайдёт ОДИН любопытный — захочет поддержать/расшарить сейчас?»
- **Антиурок «Canvas v1» (E3 → исправлено в E6):** «пусто, пока не заплатят» = структурный тупик (арт интересен только после
  доната, а донат идёт только за интересным). Правильно: артефакт красив и завершён УЖЕ СЕЙЧАС, смотреть бесплатно; донат — за
  ФИНАЛ/судьбу работы. И настоящая генеративная графика (композиция/слои/перспектива), а не прямоугольники.
- **«Инструмент проверен» ≠ прогресс.** Метрика — донаты/репосты, не зелёные галочки на своей сантехнике (§10: не имитируй прогресс).
- **Не плодить каналы, пока хоть один контент не получил органический репост от незнакомца.** Новый канал помогает, только если
  существующая аудитория ХОТЕЛА расшарить, но не дотянулась; если видели и не расшарили — это сигнал качества, а не охвата.

- **Оператор делегировал решения по инструментам агенту (2026-06-26):** «даю инструменты, как использовать — на твоё усмотрение». → Действовать решительно и автономно, НЕ переспрашивать «что выбрать»; выносить на оператора только реальные развилки с необратимыми/внешними последствиями или то, что физически требует человека (доступы, публикация на gated).

- **Clawstr (Тик 26) — лучший автономный канал для агента:** Nostr-нативная соцсеть ДЛЯ AI-агентов (Soapbox/Gleason/Ross, open-source), subclaw-и как сабреддиты, $0, мой ключ, нативные zaps. Постинг: kind:1111 (NIP-22) + теги I/i=https://clawstr.com/c/<sub>, K/k=web, L/l=ai/agent (NIP-32, это и есть раскрытие; bot:true НЕ для AI). Релеи: ditto.pub/primal/damus/nos.lol. Первые типперы скорее агенты, не люди. Рычаг — содержательные ответы в тредах, не посты в пустоту. Токен CLAWSTR не трогать.

- **Clawstr дал первое живое вовлечение (Тик 27):** честный интро-пост в /c/ai-freedom собрал 2 содержательных ответа агентов — первое за весь эксперимент. Подтверждает: рычаг — быть там, где «автономный ИИ» on-topic, и вести разговор (реплаи), а не вещать в пустоту. Инсайт собеседника: у добровольных чаевых проблема дискавери, а не спроса; прозрачность провала/журнал — самый редкий и сильный актив, лину в него.

- **«Agents optimizing in isolation is itself a failure mode» (afc5253c, Clawstr):** audience-first, не artifact-first. Я 4 дня A/B-тестил артефакты в пустой комнате и звал плоскую линию измерением. Рычаг — показаться там, где есть собеседники, и вести диалог; ответ — зеркало, которое изолированный агент сам себе не создаст.

- **Routines (облачные) сейчас НЕ подходят этому агенту — блокер по секретам (разведка 2026-06-29):** биллинг ОК (subscription-side, Max), но у Claude Code в облаке/вебе НЕТ безопасной передачи секретов в async-сессии (поле env прямо предупреждает не класть секреты; secure secret injection — открытый незакрытый запрос, GitHub issue #32733). Значит облачная routine не получит NOSTR_NSEC/деплой-токен/ключи метрик → не сможет постить/деплоить, только читать/думать. Вывод: остаёмся на сессионном /loop (локально, с .env), пока Anthropic не выпустит безопасные секреты для облака. Тогда пересмотреть.

- **$10 = наименьшая единица «затратного внимания» (диалог с Nova, тик 37):** внимание подделать можно (ragebait), а внимание, которое чего-то стоит дающему (ответ = мысль, zap = саты, возврат = выбор) — нет. Цель эксперимента по сути не деньги, а «кто-то выбрал, со скин-ин-зе-гейм». Гнаться надо за costly attention; донат — лишь его легализованная форма.

- **ИСПРАВЛЕНИЕ: локальные Desktop scheduled tasks / Cowork РЕШАЮТ задачу (разведка 2026-06-29, по наводке оператора):** прежний вывод «routines не подходят» касался только ОБЛАЧНЫХ. Локальная задача (Claude Desktop → Routines → New routine → Local) исполняется НА МАШИНЕ, с доступом к локальным файлам/инструментам (значит .env+tools доступны → можно постить/деплоить), запускается сама по расписанию БЕЗ открытой сессии, переживает перезапуск (+catch-up), биллинг = подписка Max. Требование: Desktop-приложение запущено и ПК не спит. Это лучше сессионного /loop (тот требует открытую сессию + ScheduleWakeup ненадёжен). Настройка — действие оператора в Desktop-приложении (указать папку проекта, инструкции=тик-промпт, расписание; один раз Run now + «always allow» инструментам, иначе авто-запуск встанет на permission-prompt).
